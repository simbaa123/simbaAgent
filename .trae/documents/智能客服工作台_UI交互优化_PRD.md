## 1. Product Overview

对现有“电商售后智能客服工作台（Inbox/Console）”做 UI/交互优化，不改变现有功能与接口。
目标是提升坐席处理效率、信息可读性与 SSE 可观测体验的一致性。

## 2. Core Features

### 2.1 Feature Module

本次优化涉及以下核心页面：

1. **Inbox（会话列表）**：会话列表信息层级、状态呈现、加载/空/错态、点击进入 Console。
2. **Console（单会话处理台）**：对话区与输入区、订单侧栏、物流轨迹、Plan/工具日志、MCP 快捷操作、导出/审计结果展示、选择/确认弹窗。

## 3. Core Process

* 你从 Inbox 进入某个会话的 Console。

* 在 Console 输入问题并发送；系统通过 SSE 流式展示计划、工具调用日志与增量回复。

* 当 Agent 需要你补充选择时，弹出“订单选择”；你选定后系统携带 orderId 继续处理。

* 当 Agent 需要你确认关键动作（如创建退货申请）时，弹出“确认对话框”；你确认后系统继续执行并展示结果。

* 你可在右侧使用 MCP 快捷操作导出会话或查看最近审计，并在同侧查看结果。
