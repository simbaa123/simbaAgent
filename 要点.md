## SSE 流式输出是怎么完成的（项目实现要点）

### 1) 服务端：把 HTTP 连接升级为 SSE 流

- 入口接口：`POST /api/chat/stream`
  - 位置：[server/index.ts](file:///d:/code/study/simbaAgent/server/index.ts#L118)
- 在进入业务逻辑之前，先调用 `initSse(res)` 设置 SSE 必要响应头并立刻 flush：
  - 位置：[server/sse.ts](file:///d:/code/study/simbaAgent/server/sse.ts#L3-L9)
  - 关键 header：
    - `Content-Type: text/event-stream; charset=utf-8`
    - `Cache-Control: no-cache, no-transform`
    - `Connection: keep-alive`

### 2) 服务端：用统一的 sendSseEvent 写入事件帧

- 事件帧格式（严格按 SSE 协议的 event/data + 空行分隔）：
  - 位置：[sendSseEvent](file:///d:/code/study/simbaAgent/server/sse.ts#L11-L14)

```text
event: <eventName>
data: <json-string>

```

- 项目里所有流式输出都复用这一套协议：Plan、工具日志、token 增量、弹窗事件、MCP 结果等。

### 3) 服务端：如何做到“token 级增量回复”

项目中有两种来源：

1) LangChain 模型流（USE_LANGCHAIN=1）
- 在 `runLangChainAgent` 里读取 agent 的 `streamEvents`（如果支持），捕获 `on_chat_model_stream`。
- 从 chunk 中提取 delta，并逐段发送 `assistant_delta`：
  - 位置：[server/langchainAgent.ts](file:///d:/code/study/simbaAgent/server/langchainAgent.ts#L1134-L1151)

2) 演示模式的逐字/逐段发送（USE_LANGCHAIN!=1 或部分固定文本）
- 通过 `sleep(ms)` + 分段循环调用 `sendSseEvent(..., "assistant_delta", ...)` 来模拟 token 流：
  - 位置示例：[server/index.ts](file:///d:/code/study/simbaAgent/server/index.ts#L265-L271)
  - 辅助函数：`sleep(ms)`：[server/sse.ts](file:///d:/code/study/simbaAgent/server/sse.ts#L21-L23)

### 4) 服务端：除了文本，还流式推送“可观测过程”

SSE 不只推模型文本，还推“过程事件”，让前端工作台可观测：

- `plan_update`：推送步骤数组（pending/running/done），用于右侧 Plan
  - 位置示例：[server/index.ts](file:///d:/code/study/simbaAgent/server/index.ts#L173-L173)
- `tool_call/tool_result`：每次工具调用前后各发一个事件，形成工具链路日志
  - 位置示例（LangChain 路径的统一封装）：[server/langchainAgent.ts](file:///d:/code/study/simbaAgent/server/langchainAgent.ts#L189-L190)
  - 位置示例（工具内部发事件）：[searchOrders tool](file:///d:/code/study/simbaAgent/server/langchainAgent.ts#L537-L585)
- `need_choice/need_confirm`：需要用户选择/确认时发事件，前端弹窗交互（Human-in-the-loop）
  - 位置示例（need_confirm）：[server/langchainAgent.ts](file:///d:/code/study/simbaAgent/server/langchainAgent.ts#L1060-L1072)
- `export_ready/sqlite_result`：MCP 导出/审计结果结构化推送，前端卡片/表格渲染
  - 位置示例：[server/langchainAgent.ts](file:///d:/code/study/simbaAgent/server/langchainAgent.ts#L357-L385)

### 5) 前端：如何消费 SSE（增量渲染）

前端用 fetch 的 `ReadableStream` 读取响应体，把字节流拼成文本缓冲区，再按 `\n\n` 切分事件帧：

- SSE 解析器（把文本 buffer 解析为 {events, rest}）：[parseSseTextChunk](file:///d:/code/study/simbaAgent/src/ui/pages/console/sse.js#L1-L25)
- Console 处理循环：
  - `reader.read()` → `decoder.decode()` → `parseSseTextChunk(buffer)` → 遍历 events
  - 位置：[ConsolePage.vue](file:///d:/code/study/simbaAgent/src/ui/pages/ConsolePage.vue#L149-L191)

前端按事件类型更新不同 UI：
- `assistant_delta`：把 delta 追加到最后一条 agent 消息，形成“边生成边显示”
- `plan_update`：更新 Plan 列表
- `tool_call/tool_result`：追加工具日志，同时可触发订单/物流/KB 引用卡片更新
- `final`：结束 loading/streaming 状态（本项目在服务端发送 final 后关闭响应）

### 6) 你项目 SSE 的设计亮点（面试可讲）

- “同一条 SSE 流”承载三类信息：文本流（assistant\_delta）+ 工作流状态（plan\_update）+ 工具链路（tool\_call/tool\_result）
- 前端不需要轮询，通过事件驱动实现实时 UI：消息区、Plan、工具日志、引用卡片、确认弹窗同步更新
- 通过 `inputRedacted/outputRedacted` 的结构化字段，既可展示链路又避免敏感信息泄露
